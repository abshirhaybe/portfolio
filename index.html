01 · Approach

How I work, briefly.

I treat engineering as a problem-solving job first. Every project starts with the person who'll use it, every decision earns its place, and what I ship is built to hold up in the real world, not just pass a demo.

↳ 01

User first, always.

I start with the person on the other side of the screen. What are they trying to do, where do they get stuck, and what would actually make their day easier? Pretty comes second.

↳ 02

Own it end to end.

I like owning the whole stack, from the data model and API down to the screen people use. No handoffs, no telling someone else to "make it work" and hoping for the best. I build the thing and I ship it.

↳ 03

Built to hold up.

I care about the parts users never see: handling errors and edge cases, accessible markup, and code the next person can actually read. Done means it holds up under real use, not just that it demos well.

[ Project 01 / 2026 / AI web app ]

Chair Agent: an AI that researches office chairs for you.

Chair Agent turns a vague ask like "a quiet chair for a small apartment" into a specific, reasoned pick. You tell it how you work, your budget, and your body type, and it researches real specs, prices, and reviews across the open web before it answers. It's a Next.js app built on the Anthropic API, with Claude driving a streaming agent loop that decides when to search, when to read a page in full, and when it has enough to recommend.

  • Two tools, two trust levels. A built-in web search finds candidate pages, then a custom page reader pulls the full text of the few worth opening, so the model reasons from real sources instead of guessing from snippets.
  • Source-trust rules built in. It weights independent, hands-on reviews above marketing copy, treats manufacturer pages as fact for specs but not for judgment, and says so when sources disagree.
  • Transparent, streaming UX. The answer streams in token by token while every search and page read shows up as a live status line, so you watch the research happen instead of waiting on a spinner.

[ Project 02 / 2026 / Analytics dashboard ]

CGM Dashboard: making glucose data clinically readable.

A single Python file that turns two weeks of Dexcom G7 glucose readings into the report a clinician actually reads. It computes the standard ADA 2024 metrics, GMI, Time in Range, and coefficient of variation, then lays them out the way diabetes care teams expect, so the story of someone's glucose is scannable in seconds instead of buried in a spreadsheet.

  • Clinically grounded metrics. Mean glucose, GMI, CV, and time in, below, and above range, each colored against ADA 2024 targets so anything out of range is obvious at a glance.
  • Five linked views. A 14-day trace with range bands and a zoom slider, a time-in-range donut, a 24-hour Ambulatory Glucose Profile with percentile bands, a day-by-day heatmap, and an ADA reference table.
  • Realistic simulated data. It generates 288 readings a day modeling fasting baseline, post-meal spikes, the dawn phenomenon, sensor noise, and dropouts, and swaps straight to a real Dexcom Clarity CSV export when you have one.
  • One file, zero setup. A self-contained Plotly Dash app that auto-installs its dependencies and opens in the browser, themed dark and monospace like real clinical tooling.

[ Project 03 / 2026 / Front-end redesign ]

ZIP Launchpad: rebuilding SDSU's startup incubator from scratch.

A full front-end redesign concept for ZIP Launchpad, SDSU's on-campus startup incubator. I rebuilt the whole marketing site by hand: layout, type system, color, and interactions, all in plain HTML and CSS with no frameworks and no build step. It runs from the 24/7 workspace hero down to the founder alumni and events, and it stays sharp from desktop to phone.

  • Hand-coded, no frameworks. Every section is semantic HTML and modern CSS grid, so the entire page is one file that loads instantly with nothing to build.
  • A real type and color system. Heavy Archivo display type, an SDSU-red accent, and a warm cream palette, tied together with CSS custom properties so the whole theme is consistent and swappable.
  • Responsive and accessible. The grid reflows cleanly across breakpoints, with a skip link, visible focus styles, and labelled landmarks built in from the start.
